Problem — 해결한 문제

매월 배달기사 200명의 배송건수·단가·공제항목을 엑셀로 수작업 집계하는 데 2주 전담 인력이 필요했습니다. 수동 계산으로 인한 휴먼 에러와 정산 분쟁이 반복됐고, 결과를 카카오톡·이메일로 일일이 개별 발송하는 비효율도 심각했습니다. 이 구조적 문제를 기술로 완전히 제거하는 것이 목표였습니다.

Solution — 설계 접근법

백오피스 운영 도구(Web)와 기사용 Android 앱을 Flutter 단일 코드베이스로 동시 구현하여 유지보수 비용을 최소화했습니다. 정산 상태의 복잡한 변화를 실시간 UI에 반영하면서도 성능 저하를 막기 위해 Provider 6.x 기반 상태 분리 설계를 채택했습니다. PHP REST API 레이어에서는 대량 엑셀 업로드 시 멱등성 설계(DELETE → INSERT)로 중복·재시도 상황의 데이터 오염을 방지하고, 개별 갱신에는 Upsert 패턴을 적용해 정산 데이터의 무결성과 일관성을 설계 단계에서 보장했습니다. NCP 인프라 구성부터 Apache2 웹서버·보안 설정·무중단 운영까지 시스템 전체 생명주기를 단독으로 책임지며 운영 중입니다.
